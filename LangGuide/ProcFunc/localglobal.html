Local and global variables

Variables defined inside a subroutine are called local variables. Variables defined outside a subroutine are called global variables (global relative to the subroutine).

Subroutine's parameters are considered to be its local variables.

If the name of a local variable matches the name of some global variable, then the local variable shadows the global variable, which makes it impossible to use the global variable within the subroutine. For example:

var i: real;
procedure p;
var i: integer;
begin
  // global variable i is not accessible within this procedure
  i:=5; // assignation to local variable i; ...
end;
